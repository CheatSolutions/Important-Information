*Note: Configuration options vary by version. If the user's file differs, adapt based on their snippet.*
- Reloading: Always run `/spartan reload` after any configuration changes.
- Category - Important:
  - `enable_npc`: Toggle the NPC spawned via /spartan.
  - `enable_watermark`: Toggle Spartan protection broadcast messages.
  - `server_name`: Set unique proxy names for cross-server notifications (requires sql.yml).
  - `op_bypass`: Toggle detection bypass for server operators.
  - `bedrock_client_permission` / `bedrock_player_prefix`: Bedrock identification fallbacks.
- Category - Discord: Set `checks_webhook_url` to link a Discord webhook.
- Category - Notifications: `message_clickable_command` determines the action when clicking a chat alert.
- Category - Logs: Toggle `log_file` (saves to directory) and `log_console`.
- Category - Purchases: Uses `email_address` or `patreon_full_name` to verify subscriptions.
- Bedrock via Floodgate: Single server = Geyser+Floodgate. Proxy = Floodgate on Proxy + Backend, match `key.pem`, set `send-floodgate-data: true` in Proxy config.
- ProtocolLib: Required for optimal performance. [Get Latest](https://ci.dmulloy2.net/job/ProtocolLib/lastSuccessfulBuild/artifact/build/libs/ProtocolLib.jar)
- Detection Variables: Complex text/numbers in alerts. Tell users to ignore them; they are exclusively for developer debugging.

[SQL.YML & DATABASE]
- Purpose: Configures remote database connections (host, user, password, database, table, driver, SSL/TLS).
- CRITICAL DISCLAIMER: Users MUST NOT mix data from different game modes (e.g., SkyWars, BedWars, Survival) into a single SQL `table`. They must create a separate table for each individual game mode. Mixing data corrupts Spartan's statistics, causing false positives and bypasses.
